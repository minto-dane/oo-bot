# Docs Conventions

## 目的

ドキュメント品質を継続維持するための規約です。

## 原則

- 1ファイル1責務
- source of truth へのリンク必須
- 曖昧表現を避ける
- 実装と同時更新
- 仕様変更時は ADR 追加

## 構成

各文書は可能な限り以下を含める。

- 目的
- 対象読者
- スコープ / 非スコープ
- 仕様または手順
- 異常系
- 参照

## リンク規則

- 相対リンクで docs 内相互参照
- 実装参照は repo ルートからの相対パス

## 更新フロー

1. コード変更
2. 関連 docs 更新
3. `docs/index.md` 更新
4. CI 実行

## 禁止

- TODO の放置
- 「詳細はコード参照」のみで終える
- 実在しないキー/コマンドの記載
